============================================================
0. Executive Verdict
============================================================
Current phase is NOT the time to add new physics.
Current phase IS the time to:
(1) freeze and harden v1.9g as a canonical observer build,
(2) define exact evaluation protocol and terms,
(3) run the first scale-transfer test at N=500.

Reason:
The most important unresolved question is no longer “can the system self-organize?”
That has already been demonstrated at N=200.
The central unresolved question is now:

    Is the v1.9g conclusion (hyst_0.01 => universal k=4 preference)
    a real observer law of the system, or a small-N artifact?

This is exactly the kind of question the Explainability Constitution requires us to test
before introducing any further theory or mechanism.

============================================================

============================================================
Priority 1 — CLOSE v1.9g PROPERLY
Priority 2 — RUN N=500 SCALE TEST
Priority 3 — DECIDE WHETHER OBSERVER ARCHITECTURE CHANGES ARE NEEDED
Priority 4 — ONLY THEN plan next mechanism phase

Do not reverse this order.

============================================================
2. What “closing v1.9g” should mean
============================================================
v1.9g is not fully closed merely because hyst_0.01 worked once and the report is written.
To close it as a milestone, the following must be frozen:

A. Canonical observer rule
- hyst_0.01 must be the default implementation path, not a side analysis.
- It should be exposed via one obvious config/constant, not hidden in ad hoc code.
- The code should make explicit that hysteresis applies only to the k=3 <-> k=4 boundary.

B. Canonical metric definitions
These terms must be fixed in plain text and code comments:
- dominant k*
- seed agreement
- switch rate per 100 windows
- time stability
- first-half dominant k
- second-half dominant k
- margin definition (J4 - J3 or equivalent, explicitly stated)
- tie handling
- warmup/excluded windows if any

C. Canonical output artifact
Each observer run should emit one machine-readable summary row with at least:
- N
- seed
- plb
- rate
- rule_name
- dominant_k
- switch_per100
- first_half_k
- second_half_k
- time_stable
- windows_total
- windows_used

D. Reproducibility lock
- The exact 3 x 4 x 10 panel used for v1.9g should remain reproducible.
- One canonical txt summary should be emitted automatically.

Audit note:
Until this is done, the milestone is scientifically persuasive but operationally soft.

============================================================
3. Why Priority 1 comes before N=500
============================================================
Because otherwise the N=500 test will mix two questions:
- “what happens at larger N?”
- “did our observer implementation drift between runs?”

That would violate:
Observation -> Bottleneck -> Minimal Change -> Re-observation

Therefore the observer scaffold must be frozen first.

============================================================
4. Priority 2 — The N=500 scale-transfer test
============================================================
Gemini’s proposal to test N=500 is correct and should be the next substantive experiment.

But this is not “just run a bigger graph.”
It must be framed as a hypothesis test.

Primary hypothesis to test:
H1: At larger N, island-scale information becomes less redundant, so k=3 may recover
as a non-trivial optimum in some regimes.

Null / current v1.9g carryover hypothesis:
H0: k=4 remains universally preferred under hysteresis, even at N=500.

Required discipline:
- Keep physics unchanged.
- Keep observer rule unchanged (hyst_0.01).
- Keep metric definitions unchanged.
- Change N only.

That makes the result interpretable.

============================================================
5. Minimal N=500 experimental matrix
============================================================
Recommended minimum matrix:
- N in {200, 500}
- plb in {0.007, 0.008, 0.010}
- intrusion rate in {0.0005, 0.001, 0.002, 0.005}
- seeds = 10 each minimum

This preserves direct comparability with v1.9g.

Strong recommendation:
Run 200 and 500 in the same script / same output format.
Do not create a separate one-off analysis path for 500.

============================================================
6. What must be measured at N=500
============================================================
Not just dominant k.
The following comparisons matter:

A. Observer outcome
- dominant k* distribution
- seed agreement
- switch rate
- time stability

B. Information contribution
- delta score between k=3 and k=4
- margin distribution across windows
- fraction of near-zero margins

C. Structural context change
- intrusion-hit vs non-hit drift again
- island size distribution
- number of islands / giant component behavior if relevant

D. Redundancy test
The key question is not “does k=3 ever win?”
The key question is:

    Does island-scale information become non-redundant relative to intrusion exposure?

That should be operationalized. For example:
- if k=3 wins with stable positive margins, that supports non-redundancy
- if k=4 still wins but margins collapse toward zero, that suggests weak redundancy claim
- if k=4 wins clearly with low switch rate, the small-N artifact concern weakens substantially

============================================================
7. Decision tree after N=500
============================================================
Case A: k=4 remains universal and stable
=> Conclusion:
The current observer law is not just a 200-node artifact.
=> Next step:
Do NOT add observer complexity.
Move to long-horizon robustness / larger-N replication / maybe N=1000 later.

Case B: k=3 reappears only in transition regimes with narrow margins
=> Conclusion:
Observer law is scale-sensitive, but no redesign yet.
=> Next step:
Add a “scale-sensitivity note” to the report; keep hyst_0.01 as default for N=200.
Possibly define recommended observer by N-range, but only if evidence is clean.

Case C: k=3 reappears robustly with strong margins and low switch rate
=> Conclusion:
The v1.9g “universal k=4” claim is local to N=200.
=> Next step:
Design a multiscale observer study.
Important: this still does NOT justify immediate new physics.
It just opens the next observer phase.

============================================================

can be tested cleanly.
