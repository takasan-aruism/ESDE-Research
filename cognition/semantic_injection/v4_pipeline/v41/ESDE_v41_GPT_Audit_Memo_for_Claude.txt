  2. Implementation Notes
  -------------------------

The following notes are recommended for implementation stability.

1.  Constant Propagation Speed per Wave Event

Wave propagation speed must remain constant within a single wave event.

Example:

Wave A → speed constant Wave B → speed constant

Variation between events is allowed, but not within a single event,
otherwise arrival‑time derived distances become inconsistent.

------------------------------------------------------------------------

2.  Wave Attenuation Function

To achieve the intended behavior (core destruction / peripheral
activation), an explicit attenuation rule should be defined.

Example candidates:

A_eff(h) = A0 * exp(-λh)

or

A_eff(h) = A0 / (1 + αh)

where h = graph hop distance from origin.

This ensures amplitude decays smoothly across the network.

------------------------------------------------------------------------

3.  Search Radius Definition

Activation effects should use graph hop distance, not Euclidean radius.

ESDE space is defined by topology, not fixed geometry.

------------------------------------------------------------------------

4.  Event Snapshot Logging

For analysis of structural behavior, each wave event should record:

• pre‑wave state snapshot • arrival time map • post‑wave topology state

These logs enable detection of:

• cluster death • cluster birth • migration patterns • structural
reformation
