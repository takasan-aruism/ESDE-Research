## Aruism (構造が先、意味が後)

定義 → 反転 → 非対称 → 連動 → 階層 → 命名 → 次の定義、という**繰り返し自体が「存在」と呼ばれるプロセス** (原本 行 507-511)。この繰り返しを法則化したものが Aruism。数学的に展開したのが ESDE。

## 4 層構造 (v9.11 で確立、v9.13 で方向性確定)

- **物理層** (波): Genesis 以来、ノード・リンク・位相角 θ が動的平衡。3000 本リンク、R+ 平均 6.7。常に変動。**根底**、frozen。
- **存在層** (粒子): frozenset + phase_sig で固定。label の「魂」。torque を通じて物理層に微小影響。**解放しない**。
- **認知層** (過程): 波と粒子の間に生まれるパターン。φ + attention + familiarity + B_Gen + M_c + capture。**観察のみ、介入なし**。
- **意識層** (検証、未実装): 認知層の解釈を検証する層。v10.x 以降で実装予定。**物理層・存在層・認知層に介入しない**。

**原則** (原本 行 1178-1183): 上位層は下位層を操作しない。同じ魂から経験の違いで異なるキャラクターが立ち上がる。

## 絶対遵守ルール

1. **物理層への非干渉** (原本 行 400) — 認知層・意識層から物理層を変更してはいけない。物理層は床、その上に建てる。Cognition v5.x の失敗 (「選択なき循環は洗濯機」) からの教訓。
2. **外部評価関数の禁止** — 適応度関数、loss、reward は ESDE の外側にある。内部評価のみ。
3. **既存科学オントロジーへの依存禁止** — 既知パターンへの変換は「ずれ」を招く (原本 行 556)。翻訳ではなく演繹。
4. **神の手の排除** (v9.13 で強化) — 設計者が引いた hard threshold や外部注入で構造を誘導しない。Aruism の要請は「構造が勝手に出てくる」こと。

## spatial vs structural の区別 (v9.4+ 発見)

**「見える」と「触れる」は別の知覚** (原本 行 1157-1170):

- **空間的知覚** (spatial): torus 上の BFS、固定 grid、景色。安定 (~283 ノード)
- **構造的知覚** (structural): alive link を辿る、動的、関係。激変 (17→173→47)

**L87 の例**: spatial 283、structural 47、overlap 8。**spatial の 80% は structural にならない** (Gap)。1 本のリンクの生死が数十ノードの structural 変化を引き起こす (critical transition)。

人間の比喩: 東京から見える渋谷と、毎日会う職場周辺は全く別の世界。
