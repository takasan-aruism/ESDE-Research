## v9.11 — 認知捕捉と B_Gen (2026-04-14)

### 暗号化されたパスワード (Taka 発言)

> ここで得られる数値は暗号化されたパスワードのようなもので、ほぼ一意で同じものはなく、かつ発生の確率の低さがその個体の行動を起因する原資を持つ。

B_Gen = -log10(Pbirth)。cid が誕生する確率の対数。n_core=2 の B≈12 から n_core=5 の B≈34 まで、組み合わせ項 C(5000, n) が支配する階段状バンド構造。稀な構造ほど高い B_Gen を持つ。

### 事象の定義 (Taka 発言)

> 現在ラベル化された CID は周囲のを見ることができる。事象と言っているものは、その周囲で目にする現象そのもの。その現象と自分自身が持つ情報が比較される。それによって、その情報の一致率が算出される。

E_t = 知覚圏から毎 pulse で自動抽出される事象ベクトル。設計者の恣意的定義を排除。

### 認知と意識の区別 (Taka 発言)

> 誤差の埋め合わせがまさに概念的な層 (認知寄り)。それって本当に埋め合わせできてるのか? を問えるのが意識。

認知層 = 埋め合わせ (差分の検出と記録)。意識層 = 検証 (埋め合わせの妥当性を問う)。

### 4 層の規律

```
物理層 ← 存在層 (θ torque のみ)
  ↓ (観察のみ)
認知層 (B_Gen + M_c + E_t + Δ + capture)
  ↓ (検証のみ、将来)
意識層 (認知層の解釈を検証、どこにも介入しない)
```

## v9.12 — 構造と数式の分離統合 (2026-04-16)

### 並列基準原理 (Taka 発言)

> あえて付け加えるのであれば、その基準を並列で持つことでどのようなケースにも対応できるようになる。私なんかはそうしている (A or B どちらも想定しておく。存在の対称性と存在の対等性を利用することでどっちに転んでも自分にとって OK を作り出す認知的なテクニック)

予測不能な環境 (Δ が i.i.d.) に対して、cid は複数の条件付き基準を同時保持することで対応する。予測ではなく準備。サイコロの出目は予測できないが、出目の分布は知れる。

### 構造と数式 (Taka 発言)

> 構造的に追いかける層と、数式で処理する層を分けられれば、私たちが観測者であり、その観測される対象を選ぶ立場にあることのメリットをうまく活かせる。

> 数式でいくら無限に処理を走らせたって内側で何が起こっているのかを定義できなければただのランダムである。しかし、そのランダムは場の性質上無視するわけにはいかない。

> つまり、私たちがこれはそれだ、あれはこれだ、とごちゃごちゃいうためには構造が必要なのである。この構造と数式 (おそらくは確率的発生) の二つを捉えることで神の手をさらに排除できるだろう。

- 構造 (追いかけられるもの) = frozenset, n_core, 位相 coherence, 接続トポロジー
- 数式 (場の性質) = 確率的発生、S 分布、リンク密度、Kuramoto 秩序
- S≥0.20 は構造と場の境界を設計者が引いた神の手
- Pbirth は構造 (C(N,n), r_core) と場 (ρ, S_avg) の積 → birth probability にすれば神の手を撤去できる

### S≥0.20 への態度 (Taka 発言)

> Frozenset に関して、ぼちぼち S≥0.20 と向き合うタイミングがきたように思う。なぜなら本来的にこれは神の手だから。なぜこれを使ってきたのかというと、基礎研究上の必要性があったから。しかし、動的均衡の哲学がある以上どこかで切り捨てなければならない。後回しにするほど調整が大変になるに決まってるのでさっさとやるほうがいい。

> Frozenset という概念は残しつつも、その発生に関してランダム性を持たせるのはわりとあり。つまり、どのラベルが選ばれるのかはわからないが、計算量からそれを確定していくという案。さらに、S 値は便利なので、現在使用している B_Gen に付加できるならしてしまう。
