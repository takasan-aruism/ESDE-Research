■ 点・線・形
  ノード（node）：点。世界の登場人物。
  リンク（link）：線。点と点のつながり。
  ループ（loop）：線が輪っかになった形（三角形や四角形など）。
  アイランド（island）：リンクが密に集まった塊。「村」みたいなもの。
  クラスタ（cluster）：アイランドとほぼ同義。検出方法で呼び名が変わる。
  バウンダリ（boundary）：アイランドの縁。外とつながっている点。
  インテリア（interior）：アイランドの中身。外とつながっていない点。

■ 数字（世界の体力・道の太さ）
  エネルギー E：点の元気（0〜1）。元気がないと動けない。
  リンク強度 S：線の太さ（0〜1）。太いほどよく通る。
    弱いリンク：Sが小さい（細い道）
    強いリンク：Sが大きい（太い道、反応に使える）
  共鳴 R：ループの中を通る線の「守られ度」。高いほど消えにくい。
  位相角 θ（シータ）：各点が持つ「時計の針の位置」。近い点同士が揃うと仲良くなる。

■ 2層（潜在と顕在）
  latent（潜在）L：まだ道になってない「なりうる可能性」。
  active（顕在）S：実際に存在する道（リンク）。
  realize（顕在化）：潜在Lが顕在Sになること。

■ フェーズ
  Injection：外から刺激がある時間（火入れ期間）。300ステップ。
  Quiet：外から刺激が止まる時間（静寂期間）。5000ステップ。
  Window：観測の単位。50ステップごとに1回「今どうなってる？」と見る（v4.4以降）。

■ 反応（状態）
  Dust / A / B / C：点の状態ラベル。
    Dust：ふつう（何もない）
    A/B：材料
    C：生成物（AとBがくっつくとできる）
  合成（Synthesis）：A + B → C + C
  自己触媒（Autocatalysis）：C + A → C + C
  崩壊（Decay）：Cが元気なくなるとDustに戻る

■ 観測指標
  inert collapse：リンクがゼロになって世界が沈黙すること。
  cycle（サイクル）：同じ点が Dust→A/B→C→Dust みたいに一周すること。
  k*（ケースター）：観測者が「どのくらい細かく見るか」の解像度。k*=4が最適。
  divergence（ダイバージェンス）：地方と中央の見解が食い違うこと。
  entropy（エントロピー）：多様性の指標。高い＝バラバラ、低い＝偏っている。

■ 説明可能性
  Explainability X：短い説明で「だいたいこうだ」と言える度合い。
  η（イータ、ノイズ許容）：Xがどれだけ保たれるか（0.8なら8割保つ）。

■ 3段階の閾値（v3.x以降で重要）
  S ≥ 0.30（strong）：堅い核。共鳴で守られた密な構造。「コンクリートの建物」
  S ≥ 0.20（mid）：生きた構造。入れ替わりが起きる動的な層。「木造の家」
  S ≥ 0.10（weak）：広い文脈。ほぼ全リンク。「人が歩いた跡」

■ v4系の用語
  seed（シード）：乱数の出発点。同じseedなら同じ結果が再現できる。
    「同じルールで違うサイコロの目で始めた別の宇宙」
  substrate（基盤）：ノードをグリッド状に並べた固定の隣接関係。
    wave伝播はこの基盤上を走る。active linkとは別。
  wave（波）：基盤上をBFSで広がる外部刺激。距離で減衰する。
  amplitude（振幅）：波の強さ。大きいと遠くまで届き、多くのリンクを壊す。
  density ratio（密度比）：アイランドの「内部の密さ ÷ 外部への露出」。
    高い＝内が密で外が疎＝殻ができている。
  encapsulation（エンカプスレーション）：密度比が閾値を超え続けたアイランドが
    「殻」として認定されること。中の構造を詳しく観察できるようになる。
  plasticity（可塑性）：ダメージを受けた点が一時的に新しい接続を作りやすくなること。
  hardening（硬化）：可塑的な点同士で作られたリンクが少しだけ壊れにくくなること。
  motif（モチーフ）：小さな繰り返しパターン（三角形、パスなど）。
  milestone（マイルストーン）：M1〜M5の段階的な達成指標。

■ v4.4以降の用語
  whirlpool metric（渦メトリック）：クラスターの中心ノードの「場所」で同一性を判定する方法。
    中身のノードが全部入れ替わっても、同じ場所にいれば「同じ渦」と見なす。
    「渦の中の水分子が変わっても渦は渦」という比喩。
  seen_count：whirlpoolで追跡した「この構造が何window生きたか」の数。
  Type P（persistent）：seen_count ≥ 3 のクラスター。「長生き型」。
  Type D（dense）：density ratio ≥ 1.5 のクラスター。「密度型」。
  P/D decorrelation：長生き型と密度型が別々のクラスターに現れる問題。
    同じクラスターが両方を同時に満たさないと殻にならない。
  personality signature：seen≥3に達したクラスターの内部構造のスナップショット。
    「このクラスターの顔写真」。
  deformation tolerance（変形許容）：ノードが入れ替わりながらも構造が生き延びること。
    「船のテセウス」問題の計測。
  boundary resonance（境界共鳴）：クラスターの縁に隣接する外部ノードとの位相差を測り、
    「仲間になりやすいか」を調べること。近い位相＝共鳴的、遠い位相＝非共鳴的。
  incorporation（取込）：外部ノードがクラスターの一部になること。
    v4.5a-v4.7まで全実験でゼロだった。
  accretion（降着）：外部ノードのlatent fieldを上げて、リンク化を促進する介入。
    v4.5b（1回/window）→ v4.7（毎ステップ）と進化したが効果は限定的。
  Jaccard similarity（ジャカード類似度）：2つの集合の重なり具合。
    J = (共通ノード数) ÷ (全ノード数)。0=完全別物、1=完全一致。
  relaxed_lifespan：Jaccard ≥ 0.3 で追跡した寿命。ノード入替を許容する。
  strict_lifespan：従来通りの厳密一致での寿命。
  identity_drift：クラスターのノードが部分的に入れ替わったが、同一と判定されたこと。
    「変形しながら生きている」状態の検出。
  cooling（冷却）：リンクが密な領域のsemantic pressureを弱めること。
    密な所ほどノイズが減り、位相が安定し、構造が壊れにくくなる正のフィードバック。
  cooling_factor：冷却の強さ。1.0=冷却なし。低いほどよく冷えている。
  Z-coupling（化学バレンス）：ノードの化学状態（Z）によってリンクの性質を変えること。
    Z=0（不活性）：脆い結合。すぐ切れる。「ガラスの道」
    Z=1,2（A,B元素）：普通の結合。
    Z=3（化合物C）：硬い結合。切れにくい。「鉄の道」
  phase tension（位相テンション）：A-B異種ペアの位相同期を部分的に打ち消し、
    持続的な「引っ張り合い」を作ること。極性結合のようなもの。
  motif Alpha（三角形）：3ノードの閉じたループ。最も安定な微小構造。
  motif Beta（ヒゲ付き三角形）：三角形 + degree-1の突起。反応性が高い。
  motif Gamma（四角形）：4ノードの閉じたループ。三角形より柔軟。
  Track A / Track B：並行実験の枠組み。
    Track A = terrain（地形）を大きく安定させる（cooling）。
    Track B = 質的多様性を導入する（Z-coupling）。

■ v4.8c以降の用語
  compound_restore：化合物（Z=3）リンクのdecay補正値。大きいほどリンクが長生き。
  inert_penalty：不活性（Z=0）リンクの減衰ペナルティ。大きいほど脆い。
  drift（ドリフト）：compound_restoreやinert_penaltyをシステム自身が自動調整すること。
    「パラメータを人間が決めるのではなく、システムが自分で見つける」
  α（アルファ）：drift の学習率。大きいと速く動き、小さいとゆっくり動く。
  history tensor（歴史テンソル）：各リンクが「どれだけ年をとったか」「共鳴経験」「脆さ」を
    3次元で記録するもの。リンクに「人生経験」を持たせる。
    h_age：年齢。長く生きるほど増える。
    h_res：共鳴経験。サイクルの中にいた回数。
    h_str：脆さストレス。共鳴なしに長く生きると溜まる。
  brittleness（脆性）：h_strが閾値を超え、かつSが低いリンクが「ポキッ」と折れること。
    snap（スナップ）：脆性による折れ。
  void（ヴォイド）V：リンクが死んだ跡に残る「ここに何かあった」痕跡。
    transition field（遷移場）：voidの高い場所でリンクが生まれやすくなる仕組み。
    E_ij：生まれるための障壁。degree（接続数）が多い場所ほど壁が高い。
  Π（パイ、proliferation）：「今どれだけ壊れているか」の圧力値。
    snaps / alive_links で計算。壊れるほど高い → void が活性化。
  substrate（基盤）：ノードの固定隣接関係（グリッド）。物理リンクとは別。

■ v5.0-v7.x（仮想層）の用語
  物理層（World A）：Genesis以来のルールで動く世界。ノード、リンク、化学反応。
    「現実世界の原子」に相当。固定定数（p_link_birth=0.007 など）で動く。
  仮想層（World B）：物理層の上に構築される別の世界。
    物理層を「観測」して、独自のルールで構造を作る。
    「原子の上に成り立つ化学→生物→意識」に相当。
  label（ラベル）：仮想層が「この構造に名前をつける」こと。
    物理層のクラスタやR>0 motifが繰り返し現れたら、仮想層がラベルを発行する。
    「あの三角形、また出てきた。名前つけよう」
  phase torque（位相トルク）：仮想層が物理層のθを少し回すこと。
    ラベルのphase_signature（位相の署名）に向かってθを引っ張る。
    これが物理層への唯一のフィードバック。
  semantic gravity（意味の重力）：ラベルがコアノードだけでなく、周囲のsubstrate
    隣接ノードのθも引っ張ること。構造の「影響圏」を広げる。
  territorial competition（領土競争）：複数のラベルが同じノードを取り合うこと。
    最も強いラベルがθを支配する。負けたラベルは弱くなり、死ぬ。
  recurrence（回帰）：構造が壊れて、また同じ場所に再出現すること。
  pulsing（脈動）：R>0が0→nonzero→0→nonzeroと繰り返すこと。

■ v8.x（Autonomy）の用語
  maturation（成熟）α：古いlabelの死亡閾値が下がる仕組み。
    age=20でthresholdが1/3。「年寄りは少ない収入でも生きていける」
    v4.9のh_age（リンク版、失敗）のlabel版（成功）。
  rigidity（硬直）β：古いlabelのtorqueが弱まる仕組み。
    age=10でtorqueが半分。「年寄りは周囲を変えなくなる」
    v4.9のh_res（リンク版、失敗）のlabel版（成功）。
  macro-node（マクロノード）：圧縮されたlabel。内部リンクの計算を省略。
    「速いlabel」であって「強いlabel」ではない。外との関係は通常labelと同一。
  stable trio：3つのfixed labelが位相近傍で50+ window共存するグループ。
    全48seedで出現。100%がw200まで生存。自発的に形成される上位構造候補。
  stable quad：同4つのlabel。6,383個出現。trioより多い。
  apex predator：5-nodeのlabel。他サイズを6.9倍殺す。系の「骨格」。
  displacement（追い出し）：あるlabelが死んだとき、近傍に別のlabelがいること。
    92.6%の死がこれ。「自然死」ではなく「追い出されて死ぬ」。
  relation layer：固定label間の持続的関係。trio/quadが候補。
    frozen labelを壊さずに上位構造を立てる方向。
  sharding（シャーディング）：L dict（潜在リンク場）をノードごとに分割すること。
    700万件の電話帳を5000冊に分ける。速度劣化が消える。

═══ GENESIS（創世記）═══
「何もない所から、勝手に構造が生まれるか？」

  v0.3：閉じた形（ループ）があると、構造が長生きすることを確認した。
  v0.4：全体同期じゃなく、ループ内だけの同期（局所リズム）が出ると分かった。
  v0.5：反応ルール（Dust/A/B/C）を入れたが、Quietでは回らず止まった。
  v0.6：3つの微調整を入れたが、「リンクが無いと何もできない」が確定した。
  v0.7：自動調整（controller）を入れたが、リンク再生が無いので結局止まった。
  v0.8：潜在リンク（latent）を入れて、Quietでもリンクがゼロにならず"死なない世界"になった。
  v0.9：ループ内リンクが潜在を消費して強く育ち、ついに循環（サイクル）が実測で出た。
  v1.0：A/Bの出る場所を"成長してる場所"に少し寄せて、循環が一気に出やすくなった。
  v1.1：ノイズを増やしても壊れない範囲（η）を測って、どのノイズが危険か地図化した。
  v1.2：2つのノイズ軸を同時に振って「一番うまく回る稜線（ridge）」を地図にした。
  v1.3：稜線の精密測定。点ではなくバンド（帯）だと確認。
  v1.4：30回テストして全部循環成功。ゼロ崩壊。「この世界は安定して回る」と確定。
  ── ここから「種類を増やせるか？」──
  v1.5：循環は2種類だけ（A経由とB経由）。材料が2つだから天井も2。
  v1.6：反応のエネルギー配分を変えても種類は増えなかった。頻度が変わるだけ。
  v1.7：地面に肥沃度の差を入れても種類は増えなかった。場所が変わるだけ。
  v1.8：アイランド間の微小撹乱（intrusion）を入れた。計測器のバグも4つ直した。
  v1.8-O：化学を変えず「見方」を変えた。C を構造的文脈で分類したら27種の署名が出た。
       「多様性は物の種類ではなく、観測の解像度で決まる」という大発見。
  v1.9系：観測者自身の最適解像度を自動選択する仕組み（Axiom X）を作った。
       k*=4（最も細かい文脈分類）が全条件で最適と確定。ヒステリシスで安定化。
  ── スケール拡大 ──
  N=200 → 500 → 1000 → 2000 → 5000 → 10000 まで拡大。
  k*=4は全スケールで維持。「観測者の法則はスケール不変」と確定。

═══ ECOLOGY（生態学）═══
「観測者は1人？ それとも複数？」

  v2.1：5000ノードを4地域に分けて各地域で独立観測。
     → 地方の方が中央より安定していた。中央が間違えて地方が正しい。
  v2.2：時間追跡を追加。地方は24/25 windowも同じ状態を維持する（強い持続性）。
     → 全体の40-68%のwindowで中央と地方が食い違う（divergenceが普通）。
  v2.3：食い違いは連続的なノイズではなく「エピソード」として発生する。
  v2.4：エピソードに2種類ある。短い揺れ（short-burst）と長い逸脱（long-drift）。
     → 最も多い食い違いパターンは「中央がk=3、地方全部がk=4」（中央が間違い）。
  v2.5：20 seedで確認。80%がlong_drift型。これが「この世界の普通」。
  v2.6：60 seedで3つの撹乱率で確認。70-80%がlong_drift。安定した結果。
  ═══ ECOLOGY 完了 ═══

結論：観測者は複数いる。中央は地方より不正確。食い違いが常態。

═══ COGNITION（認知）═══
「意味のある構造（概念）をトポロジーの中に作れるか？」

  ── v3.x：セマンティック注入 ──
  v3.0：2つの概念（A, B）を位相角と場所で注入。観測者は壊れなかった。
  v3.1：100 seedで確認。チェッカーボード配置（r0→A, r1→B...）が100%再現。
  v3.2：境界に微小な拡散を入れた。概念間で情報が流れ始めたが、合体はしない。
  v3.3：概念間の直接ブリッジは1 windowも持たない。位相差cos(Δθ)≈0が原因。
     → A↔Bの直接接続は物理的に不可能。油と水。
  v3.4：仲介者C（中間位相）を入れたら三者構造が85%のseedで出現。
  v3.5：decay dampening を試したが効果ゼロ。ブリッジ不安定の原因は減衰ではなく位相幾何。
  v3.6：パラダイムシフト。「橋」ではなく「流れ」として見直した。
     → 情報は境界を越えて2-3ホップ浸透する。輸送チェーン（A→C→B）は全seedで出現。
  v3.7：浸透は3-6ホップ深さで飽和。内部はほぼ全部書き換えられるが、マクロ構造は無事。
     → 「細胞膜」の比喩。中身が入れ替わっても形は残る。
  v3.8：圧力を8倍にしても浸透深度は変わらない（飽和）。80 runで崩壊ゼロ。
  v3.9：128倍まで上げてやっと2/10 seedで崩壊。深部コアは構造的に空っぽ。
     → 概念のアイデンティティは境界の動態が維持しており、内部の結束ではない。

  ── v4.0-v4.1：Wave伝播パイプライン ──
  v4.0：QwQ-32B（外部AI）とのドッキングテスト。content-physics gapを発見。
  v4.1：Hard fork。グローバル拡散を廃止し、局所的なwave伝播（BFS）に切替。
     → 基盤（substrate）上を波が走り、θシフト + リンク切断の二重効果。
     → クラスタ追跡（homeostasis, proto-reflex, proto-memory）を追加。
     → calibration結果：amplitude 2.0でもn_clusters=1, reformations=0。
        「壊すだけで再建しない」問題が確定。

  ── v4.2：適応ダイナミクス（実験的分岐）──
  v4.2：wave被害ノードにplasticity（可塑性）を付与→局所rewiring→hardening。
     → microレベルは正常動作（plasticity発動、rewiring成功、hardened links形成）。
     → macroレベルは全9 runがwave 6で崩壊。
        rewiring率が破壊の10-30%しかない。hardeningが次の波までに消滅。
     → 「災害に耐える」アプローチの限界が確定。

  ── v4.3：パラダイムシフト ──
  考え方の転換：「外から壊して耐えさせる」→「安定した殻ができたら中を育てる」
  v4.3：破壊的waveを廃止。定常的な弱いセマンティック圧力に戻す。

アリズムとESDEの関係（Takaの言葉から）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

以下はTaka（Architect）の原文を元にした、プロジェクトの哲学的基盤。

■ 存在の対称性（Symmetry of Existence）
  まず「存在」を定義する。同時にその反転（非存在）を定義する。
  この2つが対になって初めて「存在」が意味を持つ。
  例：リンクがある/ない。R>0/R=0。label が生きている/死んだ。

■ 存在の連動性（Linkage of Existence）
  ある概念の中心と、異なる概念の間に非対称性を作ることで
  一つの「存在」を定義できる。それが作用すること。
  例：labelがtorqueをかけてθが変わる → 物理層に影響する。
  仮想層の「存在」が物理層と連動して初めて意味が生まれる。

■ 存在の階層性（Hierarchy of Existence）
  定義された軸の上下に並べれば階層が生まれる。
  物理（リンク）→ 化学（Z状態）→ 構造（クラスタ）→ 仮想（label）。
  各層は下位層を観測し、上位層にフィードバックする。
  観測の粒度とタイムスケールは層ごとに異なる。

■ 軸（Axis）
  階層を貫く概念を何と呼ぶかは、途中からわかってくる。
  「生物」「意識」「集団」と呼ぶかもしれない。
  名前は最後に来る。構造が先。

■ アリズム（Aruism）→ ESDE
  上記の繰り返し（定義→反転→非対称→連動→階層→命名→次の定義...）
  そのものが「存在」と呼ばれるプロセス。
  これを哲学的にまとめたものがアリズム（Aruism）。
  アリズムから数学的に生まれたのがESDE。
  ※ Alism, Arism はLLMの誤記。正しくは Aruism。

■ 最終目標
  この繰り返し自体を法則化できないか？
  繰り返しを自律的に行うシステム = 自律的なAI = 人工知能。
  ESDEはその法則化の試み。

■ v9.2 — step-probe とtorqueの正体

  ★ 最重要発見: torqueの2相サイクル

  windowレベルでは見えなかったが、step毎に計測すると:
    torque直後：リンクが減る（攪乱）
    数step後：リンクが回復する（再構成）
    しかし回復は不完全（サイクルNETは負）

  interval=5のサイクル:
    pos0: -1.92（torque = 破壊）
    pos1: -1.00（余波）
    pos2: -1.57（余波続く）
    pos3: +0.29（回復開始）
    pos4: +1.11（回復）
    NET: -3.09（1サイクルで3本のリンク純損失）

  つまりtorqueは「安定させる力」ではなく「揺さぶって再構成を促す力」。
  
  これは設計意図と異なる。v7.0で「labelの自己維持力」として設計されたが、
  実際には「label内を揃えてlabel外を乱す」二面的な力になっていた。

■ なぜこうなったか（設計経緯）
  v7.0: torqueは「成功したら強くなる」自己強化ループだった
  v7.3: 代謝モデル（budget=1）に切り替え。自己強化ループが消えた
  v8.2: rigidity追加。古いlabelのtorqueが弱くなる
  v9.0: フィードバック乗算M追加

  力の式 sin(phase_sig - θ) は v7.0 から一度も変わっていない。
  目的が変わったのに実装が追従しなかった。

■ 方針転換

  「失敗した意図」も選択環境の一部として残す。
  labelがその圧力の下でどう振る舞うかを観測する。
  これはESDEの設計思想そのもの。
