# ESDE Genesis v1.4 — Long-Run Band Validation
## Ryzen Deployment Guide

### Prerequisites
- Python 3.8+
- NumPy, Matplotlib
- No other dependencies

### Files
```
v14_longrun.py          # Main runner (CLI interface)
genesis_state.py        # Core: node/link/latent state
genesis_physics.py      # Core: 5 forces + split step methods  
genesis_logger.py       # Metrics + crystal tracking
chemistry.py            # State transitions (A/B/C/Dust)
realization.py          # Latent → Active link conversion
autogrowth.py           # Loop-based link strengthening
```

### Quick Sanity Check
```bash
python v14_longrun.py --sanity
# Expected: ~3s, links_min > 0, "Sanity OK"
```

### Full Run (Sequential)
```bash
# All 3 points × 10 seeds = 30 runs (~2.5 hours estimated)
python v14_longrun.py
```

### Parallel Run (Recommended for Ryzen 48-thread)
```bash
# Step 1: Run baselines (must complete before experiments)
python v14_longrun.py --point P1 --seed 42  # triggers baseline computation

# Step 2: Parallel execution (20 parallel jobs on 48 threads)
parallel -j 20 python v14_longrun.py --point {1} --seed {2} \
  ::: P1 P2 P3 ::: 42 123 456 789 2024 7 314 999 55 1337

# Step 3: Aggregate results
python v14_longrun.py --aggregate-only
```

### Alternative: Manual Parallel (no GNU parallel)
```bash
# Launch in background (adjust -j count for your machine)
for P in P1 P2 P3; do
  for S in 42 123 456 789 2024 7 314 999 55 1337; do
    python v14_longrun.py --point $P --seed $S &
  done
  wait  # wait for each point's seeds to finish
done
python v14_longrun.py --aggregate-only
```

### Custom Quiet Length
```bash
# If 5000 is too slow, try 3000
python v14_longrun.py --quiet-steps 3000
```

### Output Structure
```
outputs_v14/
  baselines.json              # Cached baseline X per seed
  P1/seed_42.json             # Individual run results
  P1/seed_123.json
  ...
  v14_longrun_summary.csv     # All runs (generated by --aggregate-only)
  v14_longrun_aggregate.csv   # Per-point summary
  v14_longrun_plots.png       # Boxplots + ECDF
  v14_longrun_conclusion.txt  # Pass/fail per point
```

### Resumability
- Results are saved per seed. If a run is interrupted, re-running will
  skip completed seeds automatically.
- Delete a seed_*.json file to force re-run for that seed.

### Expected Runtime (per run)
- ~5 min per run at 5000 quiet steps (single core)
- 30 runs → ~2.5 hours sequential, ~8 min with 20 parallel jobs
