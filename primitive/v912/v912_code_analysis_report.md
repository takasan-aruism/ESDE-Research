## 調査 1: phase+r 72% の構造的原因

### 1-a. 各軸 d_* の理論的値域と正規化構造

| 軸 | 計算式 | NORM | 理論値域 | 備考 |
|---|---|---|---|---|
| d_n | \|n_core − n_local\| / 86 | 86 | [0, ∞) | n_local が 86 超なら d_n > 1.0 も可能 |
| d_s | \|S_avg − s_avg_local\| / 1.0 | 1.0 | [0, 1] | 両者とも [0,1] 内 |
| d_r | \|r_core − r_local\| / 1.0 | 1.0 | [0, 1] | 両者とも [0,1] 内 |
| d_phase | \|circular_diff(phase_sig, θ_avg_local)\| / π | π | [0, 1] | 正規化済み |

**構造的非対称性**: d_n のみ NORM=86 で割るため、実効値域が圧縮される。
d_s, d_r, d_phase はそれぞれ NORM=1.0 (または π 除算で [0,1]) のため値域を十分に使う。

### 1-b. 4 軸の実測分布 (v9.11 long, 72,264 eval pulse)

|---|---|---|---|---|

| p25 | 0.081 | 0.205 | 0.245 | 0.253 |

| 軸 | 0.25 × mean | contrib% |
|---|---|---|

> v9.11 result.md の 72% は per-subject の weighted 集計ベース。eval pulse ベースでは 62.6% だが、
> d_r + d_phase が支配的という構造は同じ。

### 1-c. 圧縮の定量

**d_n の圧縮メカニズム**:

- d_n = |n_core − n_local| / 86
- n_core ∈ {2, 3, 4, 5} (実測 max=5)
- n_local の実測: p50 での |n_core − n_local| = 0.233 × 86 ≈ **20**
- n_core=5 の場合: n_local ≈ 25 なら差 = 20 → d_n = 20/86 = 0.233 (p50 と整合)

**もし NORM_N を下げた場合の d_n 推定**:

| NORM_N | |差|=20 の d_n | d_n p50 推定 | d_r p50 との比 |
|---|---|---|---|
| 86 (現行) | 0.233 | 0.233 | 0.51× |
| 40 | 0.500 | 0.500 | 1.10× |
| 20 | 1.000 | 1.000 | 2.19× |
| 10 | 2.000 | 2.000 | 4.39× |

**結論**: NORM_N=86 は n_local の p95 ベースで設定されたが、n_core が {2..5} と小さいため、
|n_core − n_local| の典型差 (≈15-25) を 86 で割ると d_n が 0.17〜0.29 に圧縮される。
d_r (p50=0.456) や d_phase (p50=0.499) の半分以下となり、**NORM_N の過大設定が d_n 圧縮の主因**。

**d_s の圧縮メカニズム**:

- d_s = |S_avg − s_avg_local| / 1.0
- d_s の std = 0.131 (4 軸中最小) → S_avg と s_avg_local が近い値をとる
- S_avg (birth 時の平均 S) と s_avg_local (現在の局所平均 S) は同じ S 分布からサンプリングされるため差が小さい
- NORM は適切 (1.0) だが、**入力値自体の変動幅が小さい**ことが原因

### 1-d. d_r と d_phase の相関

Pearson r = **0.008** (n=72,264)

**両者はほぼ無相関**。phase と r が独立に大きな値を取って Δ を支配している。
「位相関連だから共変動」という仮説は否定された。
phase は circular_diff の特性上 [0,1] に一様に近い分布 (mean=0.500, std=0.287) となり、
r は r_core (birth 固定) と r_local (pulse 時) の差で変動幅が大きい (std=0.247)。

### 1-e. 調査 1 結論

**phase+r が Δ を支配する原因は入力値域の非対称性**:

| 要因 | 影響軸 | メカニズム |
|---|---|---|
| NORM_N=86 過大 | d_n 圧縮 | n_core∈{2..5} に対し NORM=86 は大きすぎ、d_n が 0.2 前後に固定 |
| S_avg ≈ s_avg_local | d_s 圧縮 | birth 時と現在の S が構造的に近く、差の分散が小さい (std=0.131) |
| r_core vs r_local の乖離 | d_r 拡大 | r_core は birth 時固定 (n=5 で ≈0.62)、r_local は大きく変動 |
| circular_diff の一様性 | d_phase 拡大 | phase_sig 固定 vs θ_avg_local 変動 → [0,π] に近い一様分布 |

**是正策 (仮説・要実験)**:
- NORM_N を n_local 分布の実効範囲に合わせて縮小 (例: 20〜40) すれば d_n の寄与が増大
- または軸重みを調整 (d_n, d_s の重みを上げる)
- ただし是正が capture_rate に与える影響は実験で検証が必要

## 調査 2: n_core≥6 が v9.x で出ない理由

### 2-a. label birth 条件の整理

```
                         find_islands_sets(state, s_thr=0.20)
                                   │
                                   ▼
                    S≥0.20 の連結成分 (size≥3)     ← 経路 A
                                   │
                    R>0 のリンク両端 (size=2)       ← 経路 B
                                   │
                                   ▼
                         50% overlap filter
                                   │
                                   ▼
                         phase_sig 計算 → birth
```

| 経路 | 条件 | n_core | birth 数 (v9.11) | 割合 |
|---|---|---|---|---|
| A (島) | S≥0.20 連結成分, size≥3 | 3, 4, 5 | 286 | 19.9% |
| B (R>0) | R>0 リンク両端 | 2 | 1154 | 80.1% |

### 2-b. find_islands_sets にサイズ上限はない

`find_islands_sets(state, s_thr=0.20)` の実装 (intrusion.py:23-41):
- S≥0.20 のリンクで隣接グラフを構築
- BFS で連結成分を列挙
- size≥3 のものを返す
- **サイズ上限のコードは存在しない** — size=6 以上も理論上返却可能

### 2-c. n_core 分布の実測

### 2-d. S≥0.20 閾値の設計的背景

S≥0.20 は v1.8 で確立された 3 段階閾値体系の「中間層」に位置づけられる
(参照: `docs/概念理解.md`):

| 閾値 | 意味 | メタファー |
|---|---|---|
| S≥0.30 (strong) | 堅い核。共鳴で守られた密な構造 | 「コンクリートの建物」 |
| **S≥0.20 (mid)** | **生きた構造。入れ替わりが起きる動的な層** | **「木造の家」** |
| S≥0.10 (weak) | 広い文脈。ほぼ全リンク | 「人が歩いた跡」 |

v1.8 のコード監査で、turnover が **S≥0.20 でこそ非ゼロになる** ことが確認された
(参照: `docs/ESDE_Genesis_Report.md` L278)。島検出にこの閾値が使われるのは
「代謝的に生きている構造」を捕捉するためであり、プロジェクト初期からの設計的決定。

**`find_islands_sets` の閾値変遷**:

| バージョン | s_thr | 実装 |
|---|---|---|
| v18_deploy | 0.30 固定 | `S_STRONG` ハードコード |
| v18O 以降 | 0.30 (デフォルト) | パラメータ化 (`s_thr=0.30`) |
| v9.10/v9.11 (pulse_model) | **0.20 (実引数)** | `find_islands_sets(engine.state, 0.20)` |

### 2-e. stress OFF の正確な経緯

stress OFF は「安定環境を作るためのチューニング」ではない。
**存在層が動的均衡を自律的に達成した結果、stress が不要になった**という構造的発見に基づく。

**背景** (参照: `docs/ESDE_Primitive_Report.md` L386-445):

Taka の予測: 「二重の平衡処理が不安定を生んでいる」

| 均衡機構 | 最適化対象 | 競合 |
|---|---|---|
| Stress equilibrium | リンク密度 (EMA) | ← 競合 → |
| Torque feedback (存在層) | θ 空間 (位相整合) | ← 競合 → |

2 つの機構が異なる基準で最適化し、**互いの成果を壊していた**。

**実験結果**:

| 構成変更の推移 | NET |
|---|---|
| batch gravity ON (v9.2 baseline) | -3.09 |
| → 逐次化 + age 順 + deviation | +0.08 |
| → + 500 step/window | -0.09 |
| → **+ Stress OFF** | **+0.06** |

> "torque 計算式を一切変えず、4 つの構造変更（逐次化・age 順・longer window・Stress OFF）だけで達成。"
> "Stress は仮想層が未成熟な時代の安定装置。仮想層が成熟すれば役割は縮小可能。"
> — `docs/ESDE_Primitive_Report.md` L830-836

つまり stress OFF は **存在層の自律性の証拠** であり、パラメータ変更ではなくアーキテクチャの成熟。

### 2-f. 構造的原因の分析

**原因 1: S≥0.20 部分グラフの接続性制約** (主因)

size=6 の連結成分には最低 5 本の S≥0.20 リンクが必要 (木構造の場合)。
v9.x の典型的な alive_links ≈ 2,700 本、alive_nodes ≈ 80。
全リンク密度は高い (≈0.85) だが、S≥0.20 は「生きた構造」の中間閾値であり、
全リンクの一部に限られる。Kuramoto 動力学の定常状態において
S≥0.20 部分グラフが 6 ノード以上の連結成分を形成する確率が構造的に低い。

なお、stress は S を急変させるメカニズム (`apply_stress_decay`: 高 ω・低 R のリンクを
選択的に弱化/強化) であり、v8.x で stress ON だった時期には S 分布の揺らぎが大きく
一時的に大型連結成分が出現し得た。v9.x では存在層が均衡を肩代わりしたため stress が
不要となり OFF になったが、その**副作用として** S 分布の揺らぎが減少し、
大型島の出現頻度がさらに低下したと考えられる。
これは stress OFF の目的ではなく帰結である。

**原因 2: 50% overlap フィルタの選択的抑制** (副因)

```python
for label in self.labels.values():
    overlap = label["nodes"] & cluster_nodes
    if len(overlap) > len(cluster_nodes) * 0.5:
        already_labeled = True
        break
```

大きな島ほど既存ラベルのノードと重複しやすい。
例: size=6 の島 → 既存の size=5 ラベルと 4 ノード重複 → 4 > 6×0.5=3 → **ブロック**。
小さな島 (size=3) は重複閾値が 1.5 → 2 ノード重複でもパス。
**overlap フィルタは大型島を選択的に抑制する構造**を持つ。

**原因 3: リンク形成の非空間性** (背景)

v9.x のリンクは `rng.choice(alive_arr)` (一様ランダム) で形成され、
71×71 トーラス上の空間的近接に依存しない (参照: `v911_lattice_structure_audit.md`)。
そのため「近傍ノードが密集して大型島を作る」という空間的クラスタリングが起きにくく、
S≥0.20 連結成分のサイズは接続トポロジのみで決まる。

### 2-g. 調査 2 結論

**n_core≥6 が出ない原因は 3 層の制約**:

1. **S≥0.20 部分グラフの接続性制約** (主因)
   - S≥0.20 は「生きた構造」の中間閾値 (v1.8 で確立)。定常状態で 6 ノード以上の
     連結成分が形成される確率が構造的に低い
   - stress OFF は存在層の成熟の帰結であり、その副作用として S 分布の揺らぎ減少 →
     大型島の出現頻度が追加的に低下

2. **50% overlap フィルタの選択的抑制** (副因)
   - 大型島ほど既存ラベルと重複して birth がブロックされやすい

3. **非空間的リンク形成** (背景)
   - 一様ランダムなリンク形成により空間的クラスタリングが起きにくい

**コードに明示的なサイズ上限は存在しない。** n_core≤5 は動力学的・統計的な帰結。
