------------------------------------------------------------
A1. __init__ 追加変数
------------------------------------------------------------
[ ] self._prev_territory = {}
    前windowのlabelごとのterritory_links。link_loss計算用。

[ ] self._deviation_log = []
    windowごとのdeviation summary。detail.jsonに出力。

------------------------------------------------------------
A2. Phase A: 偏りスコア計算（step()内、TORQUE前）
------------------------------------------------------------
[ ] 3つの偏り指標:

    D1: phase_drift
      |current_θ_mean - stored_phase_sig|（円周距離）
      birth時の記憶と現在のθのずれ。
      正規化: phase_drift / π → [0, 1]。重み0.5。

    D2: link_loss
      (prev_territory - current_territory) / prev_territory
      前windowからのterritory減少率。[0, 1]。重み0.3。

    D3: torque_exposure
      前windowでlabelが受けたtorque適用量。
      label["_last_torque_applied"]から取得。
      正規化: min(exposure × 100, 1.0)。重み0.2。

    combined score = 0.5 × D1 + 0.3 × D2 + 0.2 × D3

    REVIEWER: 重み（0.5/0.3/0.2）は初期設定。
    Axiom L的にはグレーだが、Phase Aの目的は
    「偏りを信号として持たせる」こと。
    重みの最適化は後。まず動くかを見る。

    REVIEWER: D3の正規化（×100）は経験的。
    torque_appliedの典型値が0.001-0.01程度のため。
    最初のrunで分布を見て調整。

------------------------------------------------------------
A3. Phase B: 局所返し（gravity_factor）
------------------------------------------------------------
[ ] gravity_factor = max(0.0, 1.0 - score)

    score=0: gravity_factor=1.0（full gravity、偏りなし）
    score=0.5: gravity_factor=0.5（gravity半減）
    score=1.0: gravity_factor=0.0（gravity停止）

    semantic gravity部で:
    grav_mag = torque_mag / nodes × gravity_factor

    REVIEWER: この返しは「偏りが大きいlabelほどgravityが弱くなる」。
    つまり「自分が偏っていると気づいたら、周囲を引っ張るのをやめる」。
    これは自然な反応。攪乱している自覚 → 攪乱を止める。

    REVIEWER: label内のtorque（core torque）は変わらない。
    変わるのはlabel外への影響（semantic gravity）のみ。
    自分の内部は保持しつつ、外への干渉を減らす。

SECTION C: 正確性検証
================================================================

[ ] phase_drift計算:
    atan2(sin_sum, cos_sum)で円周平均。
    |mean - phase_sig|の円周距離。
    alive_n < 2 → 0.0。安全。

[ ] link_loss計算:
    _prev_territoryが初回はcurrent値で初期化。
    初回loss = 0。安全。
    max(0, ...)で負（territory増加）はゼロ扱い。

[ ] torque_exposure計算:
    label["_last_torque_applied"]がない場合 → .get() で 0.0。
    初回window（まだtorque未適用）→ 0.0。安全。

[ ] gravity_factor適用:
    label内torque（L717-727）は変更なし。
    semantic gravity（L731以降）のみgf乗算。
    gf=1.0なら従来と同一動作。

[ ] _prev_territory更新:
    TORQUE前（deviation計算後）に更新。
    つまり今windowのterritory → 次windowのprev。
    タイミングは正しい。

SECTION F: KNOWN LIMITATIONS
================================================================

LIMIT 1: 偏りスコアの重み（0.5/0.3/0.2）は手動設定。

  compression_enabled=Falseの現行全runでは影響なし。
