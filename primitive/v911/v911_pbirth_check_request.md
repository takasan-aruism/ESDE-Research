# v9.11 Pbirth / B 値 計算式 検証依頼書

> 対象: `_compute_pbirth.py` で使用した B 値計算式
> 依頼元: Claude Code A (計算実行者)
> 依頼先: 独立検証者 (Claude Code B or 人間)

---

## 検証対象の式

```
rho    = links_total / C(N, 2)           ... N = 5000
Pbirth = (1 / C(N, n_core)) * rho^(n_core-1) * r_core^(n_core-1) * S_avg^(n_core-1)
B      = -log10(Pbirth)
```

---

## 検証観点

### 観点 1: 組合せ項の意味

- C(N, n_core) は「N ノードから n_core 個を選ぶ場合の数」として使っている
- **疑問**: この式は「特定の n_core 個のノードが偶然集まる確率」を表しているか?
- ラベル形成は VL の閾値超えによるもので、純粋な組合せ的ランダム選択ではない
- C(N, n_core) を使う正当性は何か? 過大評価 / 過小評価のどちらに偏るか?

### 観点 2: rho の累乗

- rho^(n_core-1) は「n_core 個のノード間に (n_core-1) 本のリンクが存在する確率」に相当
- **疑問**: 完全グラフなら C(n_core, 2) 本必要だが、n_core-1 本 (木) で良い根拠は?
- VL ラベルの形成条件 (connected component) との整合性は?
- rho^C(n_core,2) にすべきか、rho^(n_core-1) で十分か?

### 観点 3: r_core と S_avg の累乗

- r_core^(n_core-1) * S_avg^(n_core-1) の物理的意味
- **疑問**: なぜ (n_core-1) 乗なのか? 各ペア間の独立性を仮定しているか?
- Kuramoto 秩序パラメータ r_core はグループ全体の量なのに、ペア数の冪で扱っている矛盾はないか?
- S_avg もグループ平均なので同様

### 観点 4: 各因子の独立性

- 式は rho, r_core, S_avg を独立に掛けている
- **疑問**: 実際にはリンク密度が高い (rho 大) と S_avg も高くなりやすい (正の相関)
- 独立性の仮定による誤差はどの程度か?
- 実データで rho-S_avg 相関を確認すべきか?

### 観点 5: B のスケール妥当性

- 計算結果: B は 10.77 (2-node best) 〜 37.02 (5-node worst)
- **疑問**: B = 37 は「10^-37 の確率で偶然生まれる」を意味する。これは妥当か?
- 宇宙の原子数 ~10^80 と比較して、ラベル誕生に 10^-37 は過大すぎないか?
- N=5000 の系で C(5000,5) ≈ 2.6e16 が現実的な「驚き」を表すか?

### 観点 6: n_core 支配の問題

- 実データで確認: B は n_core 別のバンド構造を示し、バンド間ギャップ ~7 が支配的
- バンド内変動 (rho, r_core, S_avg 由来) は 2-4
- **疑問**: B が事実上 n_core の関数になっている場合、PI としての有用性は?
- 「同じ n_core 内で rarity に差がある」を表現できるか?

### 観点 7: 数値精度

- C(5000, 5) ≈ 2.6e16、Python int は任意精度なので overflow はない
- Pbirth ≈ 10^-37 は float64 の範囲内 (最小 ~10^-308)
- **確認**: log10 計算時に丸め誤差は無視できるレベルか?

---

## 検証手順の提案

1. 式の各項について、元の確率モデルとの対応を確認
2. n_core=2 の具体例 (lid=16) で手計算と一致するか検算
3. rho^(n_core-1) vs rho^C(n_core,2) の結果比較
4. rho-S_avg の相関係数を raw データから計算
5. B のスケールが PI として使える範囲かどうかの判断

## 参考データ

| n_core | C(5000, n_core) | log10(C) | B 実測 mean |
|---|---|---|---|
| 2 | 1.250e7 | 7.10 | 12.04 |
| 3 | 2.082e10 | 10.32 | 18.84 |
| 4 | 2.602e13 | 13.42 | 26.28 |
| 5 | 2.602e16 | 16.41 | 34.00 |

→ B_mean ≈ log10(C) + (n_core-1) * log10(1/(rho * r_core_avg * S_avg_avg)) で分解可能
