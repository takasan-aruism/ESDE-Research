### 3.3 観察 (重要)

- **phase (37-39%) + r (32-34%) で Δ の ~72% を占める**
- n 軸 (13-16%) と s 軸 (13-15%) は寄与が半分以下
- 均等重み (0.25×4) は spec 通りだが、実測では phase/r 支配
- 2 つの解釈:
  1. **設計通り**: phase/r が「同期性」の主指標、n/s は補助 → 重み均等で OK
  2. **再考の余地**: n/s の寄与が phase/r 並みになるよう重み調整すべき (例: W_n=W_s=0.4, W_r=W_phase=0.1)
- **判断保留**: Rev 2 §6-2 の初期設計は「均等」を選択しているので、
  まずは現状のまま結果を解釈し、v9.12 以降で重み調整を検討する材料とする

### 4.3 観察

- **mean capture rate ≈ 0.35-0.40** → Taka 構想「捉えにくいがゼロではない」と整合 ✓
- Δ p50 ≈ 0.37 (短/長で同等) → smoke の 0.25 から本番では若干上振れ
- Δ max が短 (1.68) で長 (1.05) より高 → short で大きな構造不一致あり (稀)
- p_capture 値域 (0.01, 0.88] で V11_P_MAX=0.9 に未到達 (設計通り、Δ>0 制約)
- cold_start 比率: short 7.5%, long 4.4% (long は tracking 50 で cold_start 比が希釈)
- 0.0 / 1.0 を取る cid も存在 (short min=0 max=1) → 少数の極端な cid あり

### 4.4 λ=2.724 の予測 vs 実測整合

- 予測: Δ p50=0.2544 → p_capture 0.5 × P_MAX = 0.45
- 実測: Δ p50=0.361, p_capture p50=0.337
- 実測 Δ が smoke より大きく本番では p_capture 平均が 0.35 前後に落ち着く
- **λ=2.724 の選択は妥当**: smoke n=150 は過小見積もりだったが、運用上は「捉えにくいがゼロではない」域

## 6. 時系列 (long, window 別 capture rate)

- window 20-69 の 50 windows を通じて rate ≈ 0.32-0.37 で推移
- 顕著な drift なし (単調増減なし)

- **M_c は birth 時固定なので、window 進行に従って Δ が少しずつ増える予想 → 緩やかな rate 低下傾向あり** (微弱だが一貫)

## 7. キー発見サマリ

1. **B_Gen バンド構造は安定** (n_core ごとに狭い分布、short/long 一致)
2. **軸寄与が偏在** (phase+r で 72-73%)、n/s は寄与小
3. **capture rate 平均 ~0.38** で設計と整合
4. **長命群は 5-core が中心、capture 困難傾向** (mean 0.31)
5. **λ=2.724 採用は運用上妥当**、本番 Δ は smoke より上振れ
6. cold_start 比率は tracking 長で希釈 (short 7.5% → long 4.4%)

## 8. 次段階への示唆 (v9.12 以降)

- **重み再考の材料**: phase+r 支配を許容するか、均等寄与を目指すか設計判断
- **長命群の追跡**: n_core=5 cids の認知捕捉推移を詳細分析すれば、
  「時間経過で M_c が現在 E_t から乖離していく」現象を直接観察できる
- **Δ max 外れ値**: short max=1.68 の pulse 群を個別分析すると構造破壊的変化が見える可能性
- **λ の実効範囲**: Δ 実分布 p50=0.37 に合わせて λ を調整すると capture rate 中央値が
  0.45 近くに移動する (smoke に引きずられた初期選択の影響)
