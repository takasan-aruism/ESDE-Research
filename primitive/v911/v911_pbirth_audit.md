# v9.11 Pbirth / B 値 試算 Audit

> 目的: v9.11 の B (Genesis Budget) を正式な確率式で試算し、実分布を確認する
> データ: v911_genesis_budget_raw.csv (1440 births, 5 seeds x 70 windows)
> 計算スクリプト: _compute_pbirth.py (audit 用、使い捨て)

---

## 1. 計算式

```
rho        = links_total / C(5000, 2)
Pbirth     = (1 / C(5000, n_core)) * rho^(n-1) * r_core^(n-1) * S_avg^(n-1)
B          = -log10(Pbirth)
```

| 因子 | 意味 | 典型値 |
|---|---|---|
| C(5000, n_core) | 5000 ノードから n_core 個選ぶ組合せ数 | C(5000,2)=1.25e7, C(5000,5)=2.60e16 |
| rho | リンク密度 = links_total / C(5000,2) | 0.00022 - 0.00025 |
| r_core | Kuramoto 秩序パラメータ | 0.08 - 1.00 |
| S_avg | ラベル内平均リンク強度 | 0.01 - 0.89 |

---

## 2. B 全体分布

| 統計量 | 値 |
|---|---|
| n | 1440 |
| min | 10.77 |
| p10 | 11.46 |
| p25 | 11.75 |
| **p50** | **12.22** |
| p75 | 12.77 |
| p90 | 27.07 |
| max | 37.02 |
| mean | 15.22 |

**特徴**: p75 = 12.77 → p90 = 27.07 で大ジャンプ。
これは n_core=2 (80%) の B が 10-14 に集中し、n_core>=3 (20%) が 18-37 に飛ぶため。
分布は連続ではなく **n_core 別の階段状バンド構造**。

---

## 3. n_core 別 B 分布

| n_core | n | 比率 | B min | B p10 | B p50 | B p90 | B max | B mean |
|---|---|---|---|---|---|---|---|---|
| 2 | 1154 | 80.1% | 10.77 | 11.41 | 12.05 | 12.70 | 13.58 | 12.04 |
| 3 | 72 | 5.0% | 18.18 | 18.37 | 18.74 | 19.49 | 20.46 | 18.84 |
| 4 | 78 | 5.4% | 25.24 | 25.57 | 26.16 | 27.07 | 28.04 | 26.28 |
| 5 | 136 | 9.4% | 32.58 | 32.97 | 33.76 | 35.53 | 37.02 | 34.00 |

**構造的ポイント**:

1. **バンド間ギャップが巨大**: n_core=2 の max (13.58) と n_core=3 の min (18.18) の間に 4.6 のギャップ
2. **バンド内レンジは小さい**: n_core=2 で 2.8, n_core=5 で 4.4 (組合せ項 log10(C(5000,n)) に比べて微小)
3. **C(5000, n_core) が支配**: 各バンドの B_mean の差 ≈ log10(C(5000,k+1)) - log10(C(5000,k))

| 遷移 | Delta B_mean | log10(C) の差 |
|---|---|---|
| 2→3 | +6.80 | log10(C(5000,3)/C(5000,2)) = +6.22 |
| 3→4 | +7.44 | log10(C(5000,4)/C(5000,3)) = +7.10 |
| 4→5 | +7.72 | log10(C(5000,5)/C(5000,4)) = +7.00 |

→ rho, r_core, S_avg の寄与はバンド内変動 (~2-4 桁) にとどまる。

## 4. 具体例: 3 個体の比較

| | 2-node 最強 | 5-node 最強 | 5-node 最弱 |
|---|---|---|---|
| lid | 16 | 59 | 295 |
| n_core | 2 | 5 | 5 |
| S_avg | 0.891 | 0.427 | 0.390 |
| r_core | 0.991 | 0.891 | 0.085 |
| rho | 0.000242 | 0.000240 | 0.000215 |
| B | **10.77** | **32.58** | **37.02** |

- 2-node 最強は S_avg=0.89, r_core=0.99 の「完璧ペア」だが B=10.77
- 5-node 最強は S_avg, r_core とも中程度だが n_core=5 なだけで B=32.58
- 5-node 最弱は r_core=0.085 (位相バラバラ) で B=37.02、同じ n_core=5 でも +4.4

→ **n_core が B を決め、r_core / S_avg は微調整**

## 5. PI 試算 (方向性のみ、パラメータ未確定)

### 案 A: PI = clamp(B_ref / B, 下限, 上限)

B_ref = p50 = 12.22 とした場合:

| ケース | B | PI |
|---|---|---|
| 2-node best | 10.77 | 1.135 |
| 2-node median | 12.05 | 1.014 |
| 5-node best | 32.58 | 0.375 |
| 5-node worst | 37.02 | 0.330 |

**問題**: n_core>=3 の PI が全て 0.3-0.7 に圧縮される。B のバンド構造が PI を事実上 n_core 別の定数にしてしまう。

### 案 B: PI = clamp(B_ref / B_within, 下限, 上限)  (n_core 別正規化)

B_within = (B - B_band_min) / B_band_range として n_core 内での相対位置を使う。
→ 同じ n_core 内での r_core / S_avg の差が PI に反映される。

### 案 C: B を n_core 別に分離し、PI 式を 2 段構成にする

- Stage 1: n_core → 基本 PI レベル (離散)
- Stage 2: バンド内 B → 微調整係数 (連続)

**判断保留**: B 分布の階段構造を見た上で、PI の設計方針を決める。

---

## 6. 観察まとめ

1. B は 10-37 の範囲で、**n_core 別の明確なバンド構造**を持つ
2. 組合せ項 C(5000, n_core) が B の主成分 (各バンド ~7 桁ジャンプ)
3. rho, r_core, S_avg はバンド内の微調整 (2-4 桁の変動)
4. 全体 p50 ベースの単純 PI では n_core の壁を越えられない
5. PI 設計には n_core 別正規化または 2 段構成が必要になる可能性が高い
