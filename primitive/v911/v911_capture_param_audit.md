## 2. V11_LAMBDA (capture 確率の指数減衰係数)

### 2.1 採用値

**V11_LAMBDA = 2.724** (Δ p50 基準)

### 2.2 決定手順

Smoke (1 seed × mat 5 × track 2 × 100 steps, λ=2.0 暫定で実行) の
pulse_log.csv から Δ 実分布を集計。
cold_start 行 (pulse_n ≤ 3) を除く eval pulse のみで分析。

Δ は λ に依存しないため、λ=2.0 で実測した分布を用いた p50 基準値の採用は有効。

### 2.3 実測 Δ 分布 (smoke eval n=150)

| 統計 | 値 |
|---|---|
| min | 0.0693 |
| p10 | 0.1312 |
| p25 | 0.1988 |
| **p50** | **0.2544** |
| p75 | 0.3345 |
| p90 | 0.3931 |
| max | 0.5252 |
| mean | 0.2645 |

### 2.4 λ 候補比較

`p_capture = V11_P_MAX × exp(-λ × Δ) = 0.5 × V11_P_MAX` となる Δ を Δ* とすると、
**λ = -ln(0.5) / Δ\***。

| 案 | 基準 Δ | 算出 λ | 設計意図 | 期待 capture_rate |
|---|---|---|---|---|
| p10 基準 | 0.1312 | 5.285 | 「最近い 10%」のみ高確率捕捉、選択性高 | ~0.30 |
| **p50 基準 (採用)** | **0.2544** | **2.724** | 中央値で p≈0.5、緩やか | ~0.48 |
| p25 基準 | 0.1988 | 3.487 | 中間 | ~0.40 |
| (現 smoke) | — | 2.000 | 暫定値、参考 | 0.593 |

### 2.5 採用理由 (Taka 構想との整合)

- **「捉えにくいがゼロではない」** という Taka の構想と整合
- p10 基準 (λ=5.285) は選択性が高すぎて、capture rate が ~0.30 まで落ち、
  「ほとんど捕捉できない」状態に近づく
- p50 基準 (λ=2.724) では Δ の中央値で p_capture ≈ 0.5、
  「半分は取り逃すが半分は取れる」というバランス
- Rev 2 §6-2 の文面は p10 基準が基本方針だが、Taka §12 回答で
  「p10 が極端に小さい場合は問題」と前置きあり。今回 p10=0.131 は
  小さくはないが p50=0.254 で λ がより穏やかな値になる方を選択

### 2.6 整合性確認

#### λ=2.0 (暫定 smoke) の実測 vs λ=2.724 (採用) の予測

| 項目 | λ=2.0 実測 (n=652, 全 pulse) | λ=2.724 予測 (eval n=150) |
|---|---|---|
| p_capture min | 0.2938 | 0.2153 |
| p_capture max | 0.8253 | 0.7451 |
| p_capture mean | 0.5284 | 0.4529 |
| p_capture @ Δ_p50 | ≈0.541 (=0.9·exp(-2·0.254)) | **0.451** ≈ V11_P_MAX × 0.5 |

- λ=2.724 では Δ=p50 で p_capture ≈ 0.451 となり、**設計値 0.45 と一致** ✓
- 値域は (0.2, 0.74] 程度に収まる見込み
- V11_P_MAX = 0.9 とのギャップ (max=0.74 < 0.9) は合理的
  (Δ=0 のときのみ 0.9 に達する)

### 2.7 P_MAX (= 0.9) の妥当性

- smoke で実測 max=0.8253 → 設計値 0.9 を超えない (Δ>0 の制約を満たす) ✓
- 「取りこぼしあり (<1.0)」の Variant A 制約を満たす ✓

### 2.8 重み (V11_W_N = V11_W_S = V11_W_R = V11_W_PHASE = 0.25) の妥当性

smoke の各軸 Δ 平均寄与:

```
※ smoke が小さい (n=150) ため、本格 run 後に再評価する
```

| 軸 | 暫定確認 | 備考 |
|---|---|---|
| d_n | 採用値 86 で正規化、smoke では概ね [0, 0.7] | 大規模 struct で 1 超え可能 |
| d_s | 1.0 で正規化、smoke では概ね [0, 0.5] | s_avg の典型値域に整合 |
| d_r | 1.0 で正規化、smoke では概ね [0, 0.7] | r_core の値域 [0,1] に整合 |
| d_phase | π で正規化、smoke では概ね [0, 1] | circular distance の最大は π |

各軸が大体同じスケールに正規化されており、均等重み (0.25) で問題なさそう。
本格 run 後に各軸の `v11_mean_d_*` 平均を比較し、特定軸が支配的なら重み調整を検討。
