    early returnの場合もデフォルト値が返る。

------------------------------------------------------------
B3. torque適用（1行変更）
------------------------------------------------------------
[ ] L595: torque_mag = energy * rigidity_factor * self._torque_multiplier

    REVIEWER: self._torque_multiplierは前windowで計算されたM。
    今windowのtorqueに前windowの状態が反映される。
    1 window遅延のフィードバック。これは意図的。
    今windowのCULL結果を今windowのtorqueに即時反映すると因果が逆転する。

------------------------------------------------------------
B4. CULL前のフィードバック計算（26行追加）
------------------------------------------------------------

[ ] died_share_sum回収: CULL判定後、削除前
    dead_labelsは既にリスト化されている（L710-716）。
    そのshareの合計を取る。

    REVIEWER: GPT要求「CULL直前固定」に準拠。
    dead_labelsはCULL判定済み（share < threshold）だが
    まだself.labelsから削除されていない。

[ ] EMA更新:
    alpha_ema = 2.0 / (tau + 1.0)
    tau = min(window_count, 20)
    
    REVIEWER: window_count < 20ではtauが小さく、
    alphaが大きくなる（直近重視）。
    20以降はalpha = 2/21 ≈ 0.095で安定。
    stress equilibriumと同一構造。

[ ] signal_ratio = died_share_sum / max(1e-6, turnover_ema)
    
    REVIEWER: turnover_emaが極小の時、ratioが巨大になる可能性。
    → clampで制限される。max(1e-6, ...)で除算安全。

[ ] M計算:
    warmup中 (window < 20): M = 1.0（EMAは更新、Mは固定）
    warmup後: M_raw = 1.0 + gamma × (ratio - 1.0)
              M = clamp(M_raw, 0.8, 1.2)

    REVIEWER: gamma=0.1, ratio=2.0 → M_raw=1.1 → M=1.1（clamp内）
    gamma=0.1, ratio=5.0 → M_raw=1.4 → M=1.2（clampで制限）
    gamma=0.1, ratio=0.0 → M_raw=0.9 → M=0.9（clamp内）
    暴走しない。

[ ] _torque_multiplierに格納。次windowのtorqueに使用。

------------------------------------------------------------
B5. return stats前のstats書き込み（6フィールド）
------------------------------------------------------------

    torque_multiplier, warmup_active, feedback_gamma

SECTION G: KNOWN LIMITATIONS
================================================================

LIMIT 1: gamma=0.1は手動設定。Axiom Lグレー。

LIMIT 2: died_share_sumは「死んだ側だけ」を見ている。
  仮想層全体の状態としては狭い。Phase 2で拡張候補。

LIMIT 3: 全labelに均等にMを適用。
  サイズ別や空間別の重みづけはPhase 2候補。

LIMIT 4: 1 window遅延フィードバック。
  今windowのCULL結果は次windowのtorqueに反映される。
  即時反映は因果逆転のため不可。

LIMIT 5: warmup 20 windowは固定値。
  系によっては20が長すぎるか短すぎる可能性。
