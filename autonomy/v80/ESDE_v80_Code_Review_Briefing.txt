2. CONVERSATION FLOW (chronological decisions)
================================================================

A. Budget=1 Metabolism (v7.3)
   Taka: "Energy is not earned. It IS. Physical stability = 1."
   Implementation: VirtualLayer allocates budget=1 among labels
   by link-influence ratio. Torque strength = share × sin(Δθ).
   Labels below half of fair share die.

B. Stress Dynamic Equilibrium (v7.2→v7.3)
   Problem: Fixed stress reference killed 300 links below baseline.
   Taka: "Reference should be system's own past."
   Implementation: EMA of link count. stress_intensity = current/EMA.
   Result: Links returned to Genesis baseline (~3100 at N=5000).

C. v7.4 Large-Scale Test
   Taka: "Scale up to N=10000, multi-seed, see what happens."
   Result: 5 seeds × 200 windows completed (39h each).
   Key finding: R+ ≈ 8 does NOT scale with N.
   Key finding: Dynamic equilibrium is seed-independent (universal).

D. Birth Pattern Discovery
   From data: 1120 labels born, 9 survived (0.8%).
   Universal law: 0 survivors born before w100 (5/5 seeds).
   5-node labels live 18× longer than 2-node.
   Oldest survivor is ALWAYS 5-node (5/5 seeds).

E. Spatial Analysis
   Labels are NOT spatial clusters. They are phase-frequency groups.
   Nodes scattered across grid but share θ.
   R>0 bridges appear BETWEEN labels, not within.

F. Macro-Node Compression Proposal (Gemini)
   Gemini proposed: compress 5-node clusters into 1 macro-node,
   scale to N=1,000,000 for emergent inter-label interaction.

G. Audit Response (Claude, this session)
   REJECTED premature compression. Reasons:
   1. Survival mechanism unknown (not share, not alignment, not phase)
   2. Cost savings minimal (2%, bottleneck is substrate scan not labels)
   3. System may not be at steady state (2 seeds show below-threshold labels)
   
   Recommended: lifecycle instrumentation first, then compression.

H. Taka's Framework
   "Points first, formulas later. Hypotheses are insufficient."
   "Designable formulas are permitted IF physically grounded."
   This aligns with: instrument → collect points → find patterns → formalize.

3. WHAT v8.0 MUST IMPLEMENT
================================================================

Goal: Add per-label lifecycle logging to virtual_layer_v2.py.
No changes to physics. No changes to stress. No new mechanics.
Pure observation. Collect the points needed for future formalization.

--- 3.1 Per-Label Lifecycle Log ---

Every label, from birth to death, must record each window:

  {
    "label_id": int,
    "window": int,
    "event": "birth" | "alive" | "death",
    "nodes": int,            # node count
    "share": float,          # budget share this window
    "alignment": float,      # cos(θ_mean - phase_sig)
    "torque_applied": float, # total torque this label applied
    "territory_links": int,  # links connected to owned nodes
    "territory_rplus": int,  # R>0 links in territory
    "territory_S_mean": float, # mean S of territory links
    "phase_sig": float,      # label's phase signature
    "nearest_label_dist": float, # phase distance to nearest label
    "n_phase_neighbors": int, # labels within 0.3 rad
  }

On death, additionally record:
  {
    "death_cause": "cull" | "no_alive_nodes" | "other",
    "share_at_death": float,
    "age_at_death": int,
  }

--- 3.2 Storage ---

In-memory list: self.lifecycle_log = []
Append one entry per label per window.
At end of run, dump to JSON alongside existing detail.json.

For 200 windows × ~20 labels = ~4000 entries. Negligible memory.

--- 3.3 Per-Window Summary (new CSV columns) ---

Add to calibrate CSV output:
  - v_oldest_age: age of oldest label
  - v_mean_age: mean age of all labels
  - v_mean_share_5node: mean share of 5-node labels
  - v_share_std: std of share distribution
  - v_births_5node: 5-node labels born this window
  - v_deaths_5node: 5-node labels that died this window

5. EXPECTED RESULTS
================================================================

From the lifecycle data, we expect to answer:

Q1: What keeps 5-node elders alive for 40-79 windows?
    Candidates to check:
    - Does share remain stable or oscillate?
    - Does alignment drift over time?
    - Do territory links grow, shrink, or cycle?
    - Is territory_rplus higher for elders?
    - Is territory_S_mean higher for elders?
    - Do elders have fewer phase neighbors (isolated niche)?

Q2: How do labels die?
    - Sudden (share drops to 0 in 1 window)?
    - Gradual (share declines over multiple windows)?
    - Triggered by newcomer birth nearby?
    - Triggered by physical event (link collapse in territory)?

Q3: Is there temporal correlation between labels?
    - Does label A's death open space for label B?
    - Do births cluster in time (multiple born same window)?
    - Do deaths cluster in time?

Q4: What distinguishes the 51 surviving 5-node from the hundreds
    of dead 5-node labels?
    - Birth conditions (what was different at birth site)?
    - Early trajectory (did survivors gain share faster)?
    - Location in phase space (were they in gaps between others)?
